🚀 ENTERPRISE MULTI-TENANT SAAS EXPENSE & INCOME MANAGEMENT SYSTEM - COMPREHENSIVE DOCUMENTATION

================================================================================
📌 PROJECT OBJECTIVE
================================================================================
Build a production-grade SaaS system with scalability, security, and maintainability.
The system is designed to handle high concurrency (10K+ users), large datasets, 
and failure scenarios like network drops or DB contention.

================================================================================
📂 CORE REQUIREMENTS (FROM ASSESSMENT)
================================================================================
- JWT Authentication: Short-lived access + Refresh tokens (rotation strategy).
- Role-Based Access: Admin, Accountant, User permissions.
- Multi-Tenant: Strict data isolation (orgId filtering).
- Financial CRUD: Transactions with Zod validation and DB transactions.
- Dashboard: Optimized aggregate queries (SUM, GROUP BY) with indexes.
- Pagination: Cursor-based for performance.
- CSV Export: Streamed large data via background jobs.
- Rate Limiting: Abuse prevention via middleware.
- Security: Helmet headers, sanitized inputs, CSRF protection.
- Frontend: Next.js App Router, React Query, Error Boundaries, Memoization.
- Backend: Layered Architecture (Controller/Service/Repository).
- Database: Normalized PostgreSQL with Prisma ORM.

================================================================================
🛠 TECH STACK
================================================================================
- Frontend: Next.js 14, Tailwind CSS, React Query, Lucide Icons, next-themes.
- Backend: Node.js, Express, TypeScript, Prisma ORM.
- Cache/Queue: Redis, BullMQ.
- Database: PostgreSQL.
- Monitoring: Prometheus, Pino (Structured Logging).
- DevOps: Docker, Docker Compose, GitHub Actions.

================================================================================
� STEP-BY-STEP IMPLEMENTATION GUIDE
================================================================================

PHASE 1: INITIALIZATION
-----------------------
1. Project structure initialized with `client` (Next.js) and `server` (Express).
2. Dependencies installed for both layers (Prisma, JWT, Zod, etc.).
3. Prisma schema designed with models for Organization, User, Transaction, and RefreshToken.

PHASE 2: CORE BACKEND (LAYERED ARCHITECTURE)
--------------------------------------------
1. Repositories: Created `AuthRepository` and `TransactionRepository` for direct DB access.
2. Services: `AuthService` (JWT logic, password hashing) and `TransactionService` (business logic, DB transactions).
3. Controllers: Handled request validation using Zod and delegated to services.
4. Middlewares:
   - `authMiddleware`: JWT verification and user injection.
   - `errorHandler`: Centralized error catching and response formatting.
   - `rateLimiter`: Request throttling.

PHASE 3: ENTERPRISE UPGRADES (SECURITY & PERFORMANCE)
-----------------------------------------------------
1. Enhanced Auth: 
   - Refresh Token Rotation with reuse detection.
   - Hashed tokens stored in DB for security.
   - Device tracking (IP & User-Agent).
   - Brute-force protection (Account locking after 5 failed attempts).
2. Multi-Tenancy: Strict `orgId` enforcement at the Repository level.
3. Performance:
   - Redis caching for Dashboard aggregates and Transaction lists.
   - Cursor-based pagination for large data.
   - Optimized indexes on `organizationId`, `date`, and `category`.
4. Observability:
   - Pino structured logging with Request ID tracing.
   - Prometheus metrics endpoint (`/metrics`).

PHASE 4: BACKGROUND JOBS & AUDITING
-----------------------------------
1. BullMQ Setup: Integrated Redis-backed queues for async tasks.
2. CSV Export: Background worker streams data to avoid memory spikes.
3. Budget Alerts: Automatic email notifications via Nodemailer when budget is exceeded.
4. Audit Logging: Every Create, Update, and Delete action is logged with user and IP metadata.

PHASE 5: FRONTEND (MODERN UI/UX)
--------------------------------
1. Setup Next.js with React Query for server-state caching.
2. Implemented Login/Register flows with proper error handling.
3. Dashboard: 
   - Interactive summary cards (Balance, Income, Expenses).
   - Filterable transaction table with Tag support.
   - Dark Mode toggle using `next-themes`.
4. Security: Secure token handling in localStorage with Axios interceptors for auto-refresh.

PHASE 6: DEVOPS & CI/CD
-----------------------
1. Dockerization: Multi-stage Dockerfiles for both Client and Server.
2. Orchestration: `docker-compose.yml` manages PostgreSQL, Redis, Backend, and Frontend.
3. CI/CD: GitHub Actions pipeline configured for:
   - Dependency installation.
   - Prisma schema generation.
   - Linting (ESLint).
   - Building (TSC & Next Build).
   - Testing (Jest).

� HOW TO RUN
================================================================================

OPTION 1: DOCKER (RECOMMENDED)
------------------------------
1. Run: `docker-compose up --build`
2. Frontend: `http://localhost:3000`
3. API Docs: `http://localhost:5000/api-docs`

OPTION 2: MANUAL
----------------
1. Server:
   - `cd server`
   - `npm install`
   - `npx prisma migrate dev`
   - `npm run dev`
2. Client:
   - `cd client`
   - `npm install`
   - `npm run dev`
