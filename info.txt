🔧 DOCKER BUILD FIX: ALPINE + PRISMA COMPATIBILITY
================================================================================
1. Root Cause:
   The error `ERROR: unrecognized option 'no-available'` was caused by an invalid flag `--no-available` in the `apk add` command. The correct flag for disabling the index cache in Alpine is `--no-cache`.

2. Corrected Dockerfile Implementation:
   ```dockerfile
   # Install dependencies needed for Prisma
   RUN apk add --no-cache openssl libc6-compat
   ```

3. Best Practices for Alpine + Prisma:
   - Use `libc6-compat`: Alpine uses `musl` instead of `glibc`. Prisma's query engine often requires `libc6-compat` to bridge this gap on Alpine.
   - Use `openssl`: Prisma requires OpenSSL to establish secure database connections.
   - Multi-stage builds: Always use multi-stage builds to keep the final production image lean.
   - Node Version: Ensure Node.js 20+ is used for Prisma v7 compatibility.

4. Verified Fix:
   - Removed `--no-available` flag.
   - Added `libc6-compat` for enhanced binary compatibility.
   - Optimized `apk` command to use `--no-cache` for smaller image layers.
