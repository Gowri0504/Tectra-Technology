🚀 Full Stack SaaS Expense & Income Management System

This document provides a comprehensive overview of the system architecture, features, implementation details, and steps to run the project.

📌 PROJECT OVERVIEW
A production-grade multi-tenant financial management system built with Next.js, Express, PostgreSQL, and Prisma.

🛠 TECH STACK
- Frontend: Next.js (App Router), Tailwind CSS, React Query, Lucide Icons.
- Backend: Node.js, Express, TypeScript.
- Database: PostgreSQL with Prisma ORM.
- Security: JWT (Access/Refresh Tokens), Helmet, Express-Rate-Limit, Bcrypt.
- Validation: Zod.
- Logging: Pino & Pino-Pretty.

✨ CORE FEATURES IMPLEMENTED
1. Multi-Tenancy: Strict data isolation using orgId on all queries.
2. JWT Authentication: Short-lived access tokens + Refresh token rotation with reuse detection.
3. Layered Architecture: Clean separation of concerns (Controller -> Service -> Repository).
4. Financial CRUD: Full transaction management with Zod validation and DB transactions.
5. Optimized Dashboard: Aggregate queries (SUM, GROUP BY) with indexed fields.
6. CSV Export: Streaming large datasets to avoid memory overload.
7. Security: Rate limiting, security headers, and input sanitization.

📂 FOLDER STRUCTURE
/server

  /src
    /controllers  - Request handling & Validation
    /services     - Business logic
    /repositories - Database abstraction

    /routes       - API route definitions
/client
  /src
    /app          - Next.js pages & layouts
    /lib          - API client (Axios with interceptors)

🚀 HOW TO RUN

1. DATABASE SETUP
- Install PostgreSQL and create a database.
- Update `server/.env` with your `DATABASE_URL`.

2. BACKEND SETUP
cd server
npm install
npx prisma generate
npx prisma migrate dev --name init
npm run dev (or: npx ts-node-dev src/index.ts)

3. FRONTEND SETUP
cd client
npm install
npm run dev

📝 IMPLEMENTATION DETAILS
- Auth Strategy: JWT tokens are stored in memory/localStorage. Refresh tokens are rotated on every use to prevent replay attacks.
- Scalability: Used indexing on `organizationId`, `date`, and `category` for fast lookups. Pagination is implemented for large datasets.
- Resilience: Centralized error handling catches all async errors and returns structured JSON responses.

✅ CI/CD (GitHub Actions)
A `.github/workflows/main.yml` should be created to:
1. Install dependencies.
2. Run Linting.
3. Run Build.
4. Verify Prisma schema.
