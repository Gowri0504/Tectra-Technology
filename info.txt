🚀 Enterprise-Grade SaaS Expense & Income Management System (UPGRADED)

This project has been upgraded to a production-ready, enterprise-level system with the following improvements:

📌 ARCHITECTURE & PATTERNS
- Layered Architecture: Strict separation between Controllers, Services, and Repositories.
- Multi-Tenancy: Automatic orgId enforcement at all levels.
- Background Processing: BullMQ + Redis for asynchronous tasks (Export, Email Alerts).

🔐 SECURITY UPGRADES
- Refresh Token Rotation: tokens are rotated on every refresh with reuse detection.
- Token Hashing: Refresh tokens are hashed using SHA-256 before being stored in the database.
- Brute-Force Protection: Login attempts are throttled, and accounts are locked after 5 failed attempts.
- Device Tracking: User-Agent and IP Address are tracked for every session.
- Input Validation: Strict Zod schemas for all API inputs.

📊 PERFORMANCE OPTIMIZATION
- Redis Caching: Frequently accessed dashboard summaries and transaction lists are cached.
- Cursor-based Pagination: Efficiently handle millions of records without performance degradation.
- N+1 Prevention: Optimized Prisma `include` and `select` queries.

📁 ADVANCED FEATURES
- Background CSV Export: Large exports are handled by background workers to prevent API timeouts.
- Audit Logging: Every Create, Update, and Delete action is logged with metadata.
- Budget Alerts: Automatic email notifications when a category budget is exceeded.
- Dark Mode: Full dark mode support in the frontend using next-themes.
- Transaction Tags: Support for multiple labels/tags per transaction.

📈 OBSERVABILITY
- Prometheus Metrics: `/metrics` endpoint for real-time system monitoring.
- Structured Logging: Request ID tracing in all logs using Pino.

⚙️ DEVOPS & CI/CD
- Dockerized: Full multi-container setup (Frontend, Backend, Postgres, Redis).
- Health Checks: Integrated health checks for database and redis services.
- Enterprise CI/CD: Comprehensive GitHub Actions pipeline for linting, building, and testing.

🚀 HOW TO RUN
1. Ensure Docker is installed.
2. Run `docker-compose up --build`.
3. Access Frontend at `http://localhost:3000`.
4. Access API Docs at `http://localhost:5000/api-docs`.
